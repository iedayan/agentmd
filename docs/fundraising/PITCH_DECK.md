## Slide 4 - Solution

AgentMD governs `AGENTS.md` execution end-to-end:

- preflight policy-as-code
- required status gates (GitHub checks)
- approvals + ownership workflows
- SSO/RBAC/compliance artifacts
- ROI/reliability analytics and ops runbook

## Slide 5 - Product Demo Flow

1. Connect repository
2. Define policy rules and required checks
3. Trigger execution
4. Preflight gate evaluates risk
5. Route approvals if needed
6. Execute and log all decisions
7. Review analytics and runbook
